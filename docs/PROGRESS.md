## Discovered facts

Empirical findings that cost someone time. Append, never delete.

- **`bun --parallel -r typecheck` (the root `bun typecheck` script) doesn't work on bun 1.3.4**
  in this environment — it prints the bun CLI help instead of running the workspace scripts.
  Ran `cd packages/<pkg> && bun run typecheck` per package instead; all four are clean. Worth
  someone re-checking whether this is a bun version issue or a `-r`-flag regression before
  relying on the root script in CI.
- **`soroban-sdk` 26.1.0's `TokenInterface::transfer` takes `to: MuxedAddress`, not `to: Address`**
  (differs from older SEP-41 examples floating around). Used the real trait signature from the
  installed crate (`~/.cargo/registry/.../soroban-sdk-26.1.0/src/token.rs`) rather than guessing.
- **A local Postgres was not running**, but Docker was available. Spun up a throwaway
  `postgres:16-alpine` container (`docker run --rm -p 5433:5432 ...`) to generate + apply the
  Prisma migration for real, then tore it down. `bunx prisma` silently no-ops (exit 0, no
  output, no migration written) when it can't find its own binary via the bun global cache —
  invoke the local `node_modules/.bin/prisma` directly instead of `bunx prisma`.
- **`packages/api`'s pre-existing test suite had latent `tsc --noEmit` failures** (unrelated to
  notes/better-auth): `hono`'s `StatusCode` type includes informational codes (e.g. `101`) that
  `c.json()` rejects (wants `ContentfulStatusCode`), and `res.json()` in the vitest files was
  typed `unknown`. Confirmed via a throwaway `git worktree` on the original commit
  (`68a6c6c`) with a fresh `bun install` — same errors, so this predates Phase 0. Fixed anyway
  (see below) since it blocked the phase's own "bun typecheck clean" exit bar and the fix was
  mechanical (a type import swap + narrow casts), not a design decision.
- Adding `SESSION_SECRET` as a hard-required env var **broke the api test suite** (vitest has no
  `.env` loaded, so `envSchema.parse(process.env)` threw). Gave it a `.default('dev-only-insecure-secret-change-me-please')`
  fallback (32+ chars, satisfies the `.min(32)`) so dev/test keep working; anything beyond local
  dev must still set a real `SESSION_SECRET` in `.env`.
